# Decisions

> ADR-style decision records.

## What's Here

Records of key architectural and design decisions.

## Format

Each decision file should include:
- **Context**: What prompted the decision
- **Decision**: What was decided
- **Consequences**: Trade-offs and implications
- **Date**: When decided

## Naming Convention

`NNNN-decision-name.md` where NNNN is sequential.

Example: `0001-use-pointer-first-approach.md`

## Current Decisions

Key decisions are captured in `planning/repo-vision.md`. As new decisions are made during implementation, add them here.
